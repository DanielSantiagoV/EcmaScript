📘 Guía Completa de ECMAScript

Una introducción fácil y visual a la evolución de JavaScript desde sus inicios hasta hoy.

¿Qué es ECMAScript?

ECMAScript es el estándar que define cómo debe funcionar el lenguaje JavaScript. Fue creado por la organización ECMA International para que los navegadores implementaran JavaScript de forma coherente.

En otras palabras, ECMAScript es como una "receta" que todos los navegadores siguen para que el código JavaScript funcione igual sin importar si usas Chrome, Firefox, Safari o Edge.

¿Cómo se usa ECMAScript?

Aunque no "instalamos" ECMAScript como tal, lo usamos todos los días al escribir código JavaScript moderno. A medida que salen nuevas versiones del estándar (cada año), navegadores y herramientas de desarrollo adoptan las nuevas funciones.

Herramientas como Babel permiten convertir código moderno a una versión compatible con navegadores antiguos.

Versiones de ECMAScript

Veamos cómo ha evolucionado ECMAScript desde sus primeras versiones hasta hoy, con ejemplos que puedes probar fácilmente.

🔹 ES1 - ES3 (1997-1999)

Primeras versiones, definieron las bases del lenguaje: variables, estructuras de control (if, while), funciones y tipos de datos.

🔹 ES5 (2009)

Ampliamente adoptada. Agregó strict mode, JSON nativo, y métodos como map(), filter() y forEach().

'use strict';
const numeros = [1, 2, 3];
numeros.forEach(n => console.log(n));

🔹 ES6 / ES2015

Una de las mayores actualizaciones: let, const, funciones flecha, clases, promesas, template literals, módulos, etc.

const nombre = "Ana";
const saludo = `Hola, ${nombre}`;
console.log(saludo);

🔹 ES7 / ES2016

Pequeñas mejoras: operador de potencia ** y método includes().

console.log(2 ** 3); // 8
console.log([1, 2, 3].includes(2));

🔹 ES8 / ES2017

Introducción de funciones asíncronas con async/await.

async function obtenerDatos() {
  const res = await fetch('/datos.json');
  const datos = await res.json();
  console.log(datos);
}

🔹 ES9 / ES2018

Agrega el operador spread/rest en objetos y Promise.prototype.finally().

const obj = {a: 1, b: 2};
const copia = {...obj, c: 3};

🔹 ES10 a ES2024

  • ES2019: Array.prototype.flat(), trimStart
  • ES2020: Optional chaining, Nullish coalescing (??)
  • ES2021: replaceAll, operadores lógicos asignables
  • ES2022: Clases con campos privados #
  • ES2023: findLast() y métodos de limpieza en objetos
  • ES2024: Propuesta de do expressions y mejoras en async context
// Optional chaining (ES2020)
const usuario = { perfil: null };
console.log(usuario.perfil?.nombre ?? "Sin nombre");

📌 Conclusión

ECMAScript es el motor detrás de JavaScript moderno. Aprender las nuevas versiones y sus funciones permite escribir código más limpio, eficiente y fácil de mantener.

No necesitas aprender todo de una vez, pero conocer las herramientas más recientes te da una gran ventaja como desarrollador web.
